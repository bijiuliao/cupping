# 咖啡網站設計規格｜以 Onyx Coffee Lab 為北極星

> 用法：把整份貼進 Claude Design（或 Claude Code）當作專案指示，
> 再附上 2–3 張 Onyx 實際頁面截圖。開場明確說：
> 「嚴格遵守這份 spec，所有顏色、字體、間距決定都要從這裡推導，
> 不要退回任何預設款。一次只做一個 section。」
> spec + 真實截圖一起餵，比只給其中一個錨得準很多。

---

## 0. 設計精神（One-liner）

一份克制、近乎信仰的編輯式電商。像一本咖啡雜誌會動：
大量留白、大字、全幅影片，每個 section 是一個「章節」。
辨識度靠**排版處理**與**真實內容密度**，不是靠裝飾。

避免：紫色、漸層、置中 hero + 三張卡片、emoji、全 pill 圓角、
彩虹式配色、填滿的大按鈕滿場飛。

---

## 1. 色票 Color tokens

克制到只有中性色 + 一點紅。紅色是標點，不是主色。

```
--ink:        #1A1A1A   /* 近黑，主要文字與深色 section（品牌名 Onyx = 黑曜石）*/
--bone:       #EEE9DF   /* 暖骨白，最常用的 section 底色 */
--warm-white: #FBFAF3   /* 更亮一階的暖白 */
--paper:      #FFFFFF   /* 純白，通常用在產品照 section */
--accent-red: #C8102E   /* 極少量使用，僅限特別版 / 聯名 / 強調（近似值，可微調）*/
```

規則：
- 一個畫面最多同時出現「一種中性底 + ink 文字」。不要多底色打架。
- 紅色整站出現頻率 < 5%。用在一兩個特別版區塊就好。
- 深色 section（ink 底、bone 字）用來做節奏斷點，別連續出現。

---

## 2. 字體 Typography

Onyx 用無襯線，不是襯線。差異來自**大小寫與字距**，不是字型花俏。

| 角色 | 處理 | 免費替代字型 |
|---|---|---|
| Display / section 標題 | 大寫、字距拉很開 `letter-spacing: 0.15–0.3em`、粗細中等 | Archivo / Space Grotesk / Geist / Public Sans |
| Body 內文 | 一般大小寫、正常字距、字級偏大好讀 | 同一套 grotesque 的 regular，或 Inter（唯一可接受用途）|
| Data / label / eyebrow | 小字、大寫、tracked、可用等寬 | IBM Plex Mono / Geist Mono |

重點規則：
- **標題一定大寫 + 拉開字距**，這是第一招牌。寧可字少也要氣派。
- 中文標題對應做法：字級拉大、字間加空（`letter-spacing`），維持同樣的「呼吸感」。
- 型號、風味、產區、烘焙資訊用等寬字，做出「實驗室 / 杯測表」的質感。
- 字級對比要大：標題很大、內文中等、label 很小。不要三種都差不多大。

---

## 3. 版面 Layout

```
全幅、上下堆疊、每個 section 邊到邊。像雜誌翻頁。

┌─────────────────────────────┐
│   [ 全幅影片 / 大圖 ]         │  ← hero：影片為主，一句 mission
│        S E C T I O N         │
│        一句短標 + 箭頭連結 →    │
└─────────────────────────────┘
┌──────────────┬──────────────┐
│  media       │  H E A D L I N E   │  ← 左右交錯
│  (影片/照片)  │  2–4 句短內文       │
│              │  Explore This →    │
├──────────────┴──────────────┤
│  H E A D L I N E   │  media   │  ← 下一段反過來
└─────────────────────────────┘
```

規則：
- **左右交錯**：media 一邊、標題+短內文+單一箭頭連結另一邊，逐段翻面。
- 每個 section 內容極簡：一個標題、2–4 句、**一個** CTA。不要塞滿。
- 留白慷慨，元素少，字大。手機版直接單欄堆疊。
- 圓角克制：接近 0 或很小。不要全 pill。

---

## 4. 動態 Motion

- **全幅、靜音、自動循環的背景影片**是預設媒材，不是點綴——這是 Onyx 的核心手感。
- 捲動觸發的淡入 / 上移 reveal。
- 橫向跑馬燈 ticker（見下方 signature）。
- hover 收斂，別讓可點元素在 hover 時淡掉或亂跳。
- 尊重 `prefers-reduced-motion`：關掉時影片給靜態海報圖。

---

## 5. 招牌元素 Signatures（記憶點）

這幾個是讓人一眼認出「這是 Onyx 那派」的東西，挑 2–3 個做滿：

1. **拉開字距的大寫標題**（最強招牌）。
2. **`|` 分隔的風味描述**，粗體：`Apple | Baker's Chocolate | Orange | Silky`。
3. **全幅自動播放影片** section。
4. **橫向跑馬燈**：例如「Now Listening」樂團名、「Trending Coffees」豆名，帶出品牌個性。
5. **透明度數據**：杯測分數、烘焙追蹤、生產者 spotlight（真名、農場、處理法、品種）。
6. **獎項 / 媒體引言牆**：用密度堆出可信度（年份 + 名次的長清單）。

---

## 6. 品牌口吻 Voice

- 近乎信仰、使命感重：`Join Our Pilgrimage`、`quality, truth and accountability`、`Never Settle for Good Enough`。
- **生產者優先的敘事**：講真實的人、農場、處理法、品種，教育性 + 透明。
- 自信但不推銷，讓工藝與獎項自己說話。
- 每段內文短（2–4 句），一個清楚 CTA。
- CTA 用**文字 + 箭頭**（`Explore This Offering →`），不是滿場填色大按鈕。

---

## 7. 關鍵提醒（別複製成空殼）

Onyx 的版面能成立，是因為底下有**真實而深的內容**：生產者故事、杯測數據、
影片、獎項。空版面套上這套排版只會變好看的殼。

所以餵給 Claude Design 時，**用你自己的真材料**：真實豆名、風味、
產區、處理法、你田野裡的產地故事與照片。內容夠真，這套設計語言才會活。
